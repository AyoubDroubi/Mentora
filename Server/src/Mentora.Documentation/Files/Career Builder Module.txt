Software Requirements (SWR)
Career Builder Module
1. Overview
The Career Builder module helps users define a clear career direction based on self-assessment and AI
analysis. It transforms a career quiz into structured career plans, tracks progress, and visualizes skill
development.
2. Global Rules
All endpoints require JWT authentication. Data is user-scoped, persisted using GUIDs, timestamps are
stored in UTC, and only one active career plan may be worked on at a time.
3. Career Plan
AI-generated career plans are stored with statuses: Generated, InProgress, Completed, or Outdated. Each
plan contains ordered steps and a calculated progress percentage.
4. Career Progress
A read-only view that visualizes progress across steps. Progress is calculated automatically based on
completed skills and propagated to the plan level.
5. Skills Snapshot
Displays required technical and soft skills as checklists. Each skill has a status: Missing, InProgress, or
Achieved, and updates plan and step progress automatically.
6. Career Quiz
A standardized diagnostic quiz that helps users understand themselves and generates a personalized
career plan using AI. Quiz statuses include Drafted, Completed, and Outdated.
7. AI Integration Rules
AI is invoked only after quiz submission. The response must be structured and parsed into plans, steps,
and skills.
8. Backend Summary
The backend must handle AI parsing, progress propagation, state management, and secure data access
for all Career Builder features.

Software Requirements (SWR)
Module: Career Builder
1. Overview

The Career Builder module helps users define a clear career direction based on self-assessment and AI analysis.
It transforms a career quiz into structured career plans, tracks progress, and visualizes skill development.

The module consists of four core features:

Career Plans

Career Progress

Skills Snapshot

Career Quiz

All data is user-specific, persisted in the database, and served via authenticated APIs.

2. Global Rules (Applies to Career Builder)
2.1 Authentication

All endpoints must require JWT authentication.

UserId must be extracted from the token.

Users may only access their own career data.

2.2 Data Integrity

All primary keys use GUID.

Timestamps stored in UTC.

Soft delete is recommended for plans and quiz attempts.

Only one active Career Plan may be worked on at a time.

3. Feature #1 — Career Plan
3.1 Objective

Display AI-generated career plans and allow users to follow them step by step while tracking progress.

3.2 Career Plan Statuses

Each Career Plan must have one of the following statuses:

Generated
Plan was created by AI but user has not started working on it.

InProgress
User has started working on the plan.

Completed
All steps and required skills have been completed.

Outdated
The quiz was re-taken or modified after this plan was generated.

3.3 Functional Requirements

FR-CP-01
The system shall store all AI-generated career plans for the user.

FR-CP-02
Each plan shall be displayed in a list with:

Title

Status

Progress percentage

Creation date

FR-CP-03
Each plan shall be divided into ordered Steps.

FR-CP-04
Each Step shall have:

Step name

Description (optional)

Progress percentage

FR-CP-05
Clicking a plan opens a detailed page showing its steps and progress line.

3.4 Data Model — CareerPlan
Field	Type
Id	Guid
UserId	Guid
Title	string
Summary	string
Status	enum (Generated / InProgress / Completed / Outdated)
ProgressPercentage	int (0–100)
CreatedAtUtc	DateTime
IsDeleted	bool
3.5 Data Model — CareerStep
Field	Type
Id	Guid
CareerPlanId	Guid
Name	string
Description	string
OrderIndex	int
ProgressPercentage	int
4. Feature #2 — Career Progress
4.1 Objective

Provide a read-only view that visualizes the user’s progress in the active career plan.

4.2 Functional Requirements

FR-PR-01
The system shall display the active career plan progress.

FR-PR-02
Each step shall display:

Step name

Step progress percentage

FR-PR-03
Career Progress shall be calculated automatically based on completed skills.

FR-PR-04
The user shall not be able to manually modify progress from this page.

4.3 Progress Calculation Rules

Step progress is derived from the completion state of related skills.

Career Plan progress is the average of all step progress values.

Progress values are integers (0–100).

5. Feature #3 — Skills Snapshot
5.1 Objective

Display required skills generated by AI and allow the user to track their completion status.

5.2 Skill Categories

Skills are categorized as:

Technical Skills

Soft Skills

5.3 Skill Statuses

Each skill must have one of the following statuses:

Missing

InProgress

Achieved

5.4 Functional Requirements

FR-SK-01
The system shall display all required skills for the active career plan.

FR-SK-02
Skills shall be displayed as a checklist.

FR-SK-03
The user shall be able to update the status of a skill.

FR-SK-04
Updating a skill status shall automatically update:

Step progress

Career Plan progress

Career Progress page

FR-SK-05
Skill completion percentages shall be displayed per category (as shown in mock data).

5.5 Data Model — Skill
Field	Type
Id	Guid
Name	string
Category	enum (Technical / Soft)
5.6 Data Model — CareerPlanSkill
Field	Type
Id	Guid
CareerPlanId	Guid
SkillId	Guid
Status	enum (Missing / InProgress / Achieved)
6. Feature #4 — Career Quiz
6.1 Objective

Help users understand themselves and make informed career decisions instead of guessing paths randomly.

6.2 Quiz Statuses

Each quiz attempt must have a status:

Drafted — quiz started but not submitted

Completed — quiz submitted successfully

Outdated — a newer quiz attempt exists

6.3 Functional Requirements

FR-QZ-01
The quiz questions shall be standardized for all users.

FR-QZ-02
The system shall allow users to save a quiz attempt as Draft.

FR-QZ-03
Submitting the quiz sets the status to Completed.

FR-QZ-04
A completed quiz enables the “Generate Career Plan” action.

FR-QZ-05
If a new quiz is submitted:

The previous quiz becomes Outdated.

Any career plan linked to the previous quiz becomes Outdated.

FR-QZ-06
AI-generated data shall be distributed across:

Career Plans

Career Steps

Skills Snapshot

6.4 Data Model — CareerQuizAttempt
Field	Type
Id	Guid
UserId	Guid
AnswersJson	string
Status	enum (Drafted / Completed / Outdated)
CreatedAtUtc	DateTime
7. AI Integration Rules

AI is called only after quiz submission.

Prompt includes:

User quiz answers

Career constraints (time, goals, obstacles)

AI response must be returned in a structured format (JSON).

Parsed data populates:

CareerPlan

CareerSteps

Skills

8. Required API Endpoints (Summary)
Quiz

GET /api/career-quiz/questions

POST /api/career-quiz/save-draft

POST /api/career-quiz/submit

GET /api/career-quiz/latest

Career Plan

POST /api/career-plans/generate

GET /api/career-plans

GET /api/career-plans/{id}

Skills

GET /api/career-plans/{id}/skills

PATCH /api/career-plans/{id}/skills/{skillId}

Progress

GET /api/career-progress/active

9. Relationship with Study Planner Progress (Important)

Career Builder progress feeds into Study Planner main page progress (0–100).

Study Planner progress is calculated independently but may be extended later to include Career progress.

10. Summary for Senior Backend Developer

This module requires:

Strong state management for Quiz and Career Plans

AI-to-DB structured parsing

Automatic progress propagation from Skills → Steps → Plan

Clear enforcement of Outdated vs Active entities

Fully read-only Career Progress visualization
